# 🚀 FitBridge - PRD (Product Requirements Document)

> 본 문서는 FitBridge MVP의 제품 요구사항을 정의합니다.
> 팀 협업 규칙 및 역할 분담은 [CLAUDE.md](./CLAUDE.md)를 참고하세요.

## 1. 🎯 제품 목표

*   **최우선 목표:** 외국인 여행자의 '언어 장벽'과 '음식 선택의 불안감(알레르기, 맵기)' 해소에만 집중한다.
*   **MVP 절대 원칙:** 아래 정의된 **3가지 핵심 기능** 외의 추가 아이디어는 모두 'Nice-to-have'로 백로그에 넘긴다. 기한 내 핵심 기능 완수에 총력을 다한다.

---

## 2. 🎯 MVP 핵심 기능 3가지 (Core Features)

### ① 나의 입맛 프로필 (Taste Profile)
*   **기능 요약:** 앱 최초 실행 시 사용자의 식습관 데이터를 수집하여 필터링 기준으로 삼는다.
*   **상세 스펙:**
    *   매운맛 선호도 설정 (0~3단계)
    *   알레르기 및 식이 제한(비건, 할랄 등) 설정
*   **목적:** 이후 메뉴를 분석할 때 사용자 맞춤형 경고를 띄워주는 개인화 기준점 확보.

### ② AI 스마트 메뉴판 분석 및 번역
*   **기능 요약:** 카메라로 촬영한 한국어 메뉴판을 다국어로 번역하고 구조화된 데이터를 제공한다.
*   **상세 스펙:**
    *   Gemini Vision API (Vercel Serverless 연동) 활용
    *   메뉴명(원문/발음/번역), 주요 식재료, 먹는 방법(Eating Method) 제공
    *   **[핵심]** 사용자의 '입맛 프로필'과 대조하여 위험 요소(알레르기, 초과 맵기) 발견 시 시각적 경고 배지(🌶️, ⚠️) 즉시 노출.

### ③ 알레르기 알림 카드 (Show to Staff)
*   **기능 요약:** 언어가 통하지 않는 식당 직원에게 직접 보여주어 소통하는 텍스트 카드.
*   **상세 스펙:**
    *   "🚫 저는 [해당 재료] 알레르기가 있습니다. 절대 빼주세요." 형태의 명확한 한국어 텍스트와 외국어 병기.
    *   직원이 바쁜 상황에서도 단번에 인지할 수 있도록 직관적이고 강력한 UI/색상 적용.

---

## 3. 📦 Nice-to-have (초기 MVP 제외 대상)

다음 기능들은 서비스 완성도를 높여주지만, 초기 MVP 집중을 위해 추후 릴리즈로 미룬다.

*   **UGC 및 리뷰 시스템:** 사용자가 직접 음식 사진을 갤러리에서 업로드하고 맵기 평가를 남기는 기능.
*   **게이미피케이션:** 리뷰 작성 시 '전포 미식가' 배지 부여 등 사용자 활동 보상.
*   **커스텀 지도 탐색 (Mapbox/Google Maps):** 주변 식당 위치 표시 및 탐색 기능. (초기에는 '탐색'보다 '분석/소통 도구'에 집중)
