## 1. 📋 팀 공통 규칙 (Team Guidelines)

*   **최우선 목표:** 외국인 여행자의 '언어 장벽'과 '음식 선택의 불안감(알레르기, 맵기)' 해소에만 집중한다.
*   **MVP 절대 원칙:** [PRD.md](./PRD.md)에 정의된 **3가지 핵심 기능** 외의 추가 아이디어는 모두 'Nice-to-have'로 백로그에 넘긴다. 기한 내 핵심 기능 완수에 총력을 다한다.
*   **UI/UX 원칙:** 심미성보다는 **직관성**과 **안전(Safety)**이 최우선이다. 알레르기 경고, 매운맛 등 생존 및 안전과 직결된 정보가 가장 먼저 눈에 띄도록 디자인한다.
*   **협업 방식:** 기능 추가나 변경이 필요할 경우, 코드를 먼저 수정하지 않고 반드시 [PRD.md](./PRD.md)를 기준으로 팀 합의를 먼저 거친다.

---

## 2. 👥 팀 역할 분담 (Feature Ownership)

MVP 핵심 기능 3가지에 맞춰 팀원 3명이 각각 하나의 기능을 처음부터 끝까지(기획-구현-테스트) 책임진다.

| 담당 | 기능 | 범위 |
|---|---|---|
| **팀원 A** | ① 나의 입맛 프로필 (Taste Profile) | 최초 실행 온보딩 UI, 매운맛/알레르기/식이 제한 설정 및 저장 로직 |
| **팀원 B** | ② AI 스마트 메뉴판 분석 및 번역 | Gemini Vision API 연동(`api/analyze-menu.js`), 번역 결과 구조화, 위험 요소 경고 배지 로직 |
| **팀원 C** | ③ 알레르기 알림 카드 (Show to Staff) | 다국어 텍스트 카드 UI, 직관적 경고 색상/디자인 |

*   기능 간 의존성(예: ②가 ①의 프로필 데이터를 사용)이 있으므로, 데이터 스키마(입맛 프로필 형식 등)는 착수 전 3명이 함께 정의한다.
*   기능 경계를 넘는 작업(공통 컴포넌트, 배포 설정 등)은 담당자를 정하지 않고 그때그때 협의한다.
